## 1. Alineación con Marcos de Inteligencia de Amenazas (CTI)

### MITRE ATT&CK

- **Tácticas:** Persistence (TA0003)
    
- **Técnica:** Create Account: Local Account (T1136.001)
    
- **Plataforma:** Windows
    

### Lockheed Martin Cyber Kill Chain

- **Fase:** Installation / Actions on Objectives
    
- **Contexto de Fase:** El adversario ya ha superado las fases de entrega y explotación (posee acceso con privilegios) y está modificando la configuración del sistema operativo para asegurar su permanencia y redundancia de acceso (Installation) antes de ejecutar movimientos laterales o exfiltración.
    

### Modelo Diamante (Diamond Model)

- **Adversario:** Actor de amenazas no identificado (fase de post-compromiso).
    
- **Capacidad:** Uso de binarios nativos del sistema (LOLBins - `net.exe`) para la manipulación de la base de datos SAM y grupos de seguridad local.
    
- **Infraestructura:** Host local comprometido y credenciales de administrador legítimo vulneradas.
    
- **Víctima:** Endpoint Windows 10 (DESKTOP-HE2FRPA).

## 2. Escenario e Hipótesis

**Escenario Base:** El equipo de respuesta recibe una alerta de bajo nivel sobre actividad inusual en el host DESKTOP-HE2FRPA a las 08:12 h. Se detecta que una cuenta de administrador legítima ejecutó comandos anómalos, sin escalar automáticamente en el SIEM.

**Hipótesis de Caza (Hunt Hypothesis):** "Un adversario con acceso inicial ha creado una o más cuentas locales no autorizadas y las ha añadido al grupo Administrators para garantizar persistencia redundante en el sistema comprometido."

**Infraestructura de Análisis:**

- Host: DESKTOP-HE2FRPA (Windows 10)
    
- SIEM: Elasticsearch 8.17.10 + Kibana
    
- Índice: filebeat-8.14.0 -*
    
- Dataset: t1136_windows_logs.json

## 3. Diccionario de Eventos Críticos (Telemetría)

- **4720 (User Account Created):** Evento principal de detonación. Identifica el sujeto creador y el nombre de la nueva cuenta.
    
- **4722 (User Account Enabled):** Evento subsecuente al 4720. Confirma la activación y operatividad de la cuenta.
    
- **4732 (Member Added to Local Group):** Indicador de escalada. La cuenta es agregada al grupo Administrators (S-1-5-32-544).
    
- **4688 (Process Created):** Visibilidad de ejecución de comandos en línea (`net.exe /add`). Requiere auditoría de creación de procesos.
    
- **4624 (Successful Logon):** Validación de acceso. Un Logon Type 3 (acceso por red) desde una IP externa es un indicador crítico de compromiso.
    
- **4625 (Failed Logon):** Intentos de acceso fallidos. Indicador de posible reconocimiento interno o fallos en herramientas de movimiento lateral.

## 8. Gestión de Falsos Positivos (False Positives)

Antes de escalar una alerta basada en esta hipótesis, el analista debe descartar las siguientes actividades administrativas legítimas:

- **Sistemas de Aprovisionamiento Automatizado:** Herramientas como SCCM (System Center Configuration Manager), Ansible o scripts de automatización de IT que crean cuentas locales temporales para mantenimiento.
    
- **Soluciones LAPS (Local Administrator Password Solution):** Aunque LAPS rota contraseñas y no suele crear cuentas nuevas constantemente, configuraciones erróneas en las GPOs podrían generar ruido en el Event ID 4732.
    
- **Instalación de Software Legítimo:** Ciertos instaladores de bases de datos (ej. SQL Server, Oracle) o servicios de respaldo (ej. Veeam) pueden crear cuentas de servicio locales y asignarles privilegios de administrador durante su despliegue inicial.

## 9. Puntos de Pivote y Siguientes Pasos (Next Steps)

Si la hipótesis se confirma y la actividad se clasifica como maliciosa (True Positive), el investigador debe pivotar hacia las siguientes áreas de telemetría:

- **Movimiento Lateral:** Filtrar logs de red y eventos de autenticación (EID 4624 Logon Type 3) originados desde el host comprometido (DESKTOP-HE2FRPA) hacia el resto del dominio, utilizando la cuenta recién creada.
    
- **Ejecución Remota:** Buscar evidencia de uso de herramientas como `PsExec`, `WMI` (Eventos de Microsoft-Windows-WMI-Activity) o `WinRM` asociadas a la nueva cuenta.
    
- **Exfiltración y Acceso a Datos:** Revisar el acceso a recursos compartidos (EID 5140) y la ejecución de herramientas de compresión/archivado (ej. 7zip, WinRAR) en el host comprometido.

## 10. Recomendaciones de Mitigación

Para reducir la superficie de ataque frente a esta técnica de persistencia, se recomiendan las siguientes acciones preventivas:

- **Restricción de Grupos Locales mediante GPO:** Implementar políticas de "Restricted Groups" (Grupos Restringidos) en Active Directory para forzar que el grupo de Administradores Locales solo contenga cuentas explícitamente autorizadas, revirtiendo cualquier adición no autorizada automáticamente.
    
- **Implementación de LAPS:** Asegurar que la cuenta de administrador local por defecto (SID 500) tenga una contraseña única, compleja y rotatoria por cada endpoint.
    
- **Principio de Menor Privilegio:** Auditar y retirar derechos de administrador local a los usuarios regulares. Si un atacante compromete a un usuario estándar, no podrá ejecutar `net localgroup administrators /add`.
